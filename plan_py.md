## План: Реалистичный симулятор удара мяча о многослойную поверхность (Python + Arcade)

### 1. Summary

Приложение реализовано как настольное приложение на **Python 3.13** с использованием `**arcade` (последней версии)**.
UI/рендеринг выполняются через механизмы `arcade` и (по возможности) `arcade.gui`, без привязки к браузеру.

Цель: реалистично моделировать фазу контакта и короткий пост-контактный полет, корректно показывая:

- движение мяча по заданному углу (без «горизонтального по умолчанию»),
- визуально заметное вращение (метка на мяче + стрелка направления),
- многослойную структуру поверхности в разрезе (динамическая деформация не визуализируется),
- управляемое трение (локальные `mu_s`, `mu_k` + глобальный множитель),
- центрированную по X поверхность; удар происходит в центре кадра по X.

Физический уровень: максимально реалистичный численный ядро v1 (сложная численная модель допускается).

### 2. Реализация (план для Python/Arcade)

Проект будет реализован как набор модулей Python с разделением ответственности по смыслу (численное ядро, рендер, UI, валидация).

Рекомендуемая структура (пример):

1. `main.py`: точка входа (создание окна, запуск `arcade`).
2. `app/window.py`: главный класс окна/представления (`arcade.Window` / `arcade.View`), обработчики `on_draw`, `on_update`, `on_mouse_press`.
3. `config/constants.py`: численные параметры и константы устойчивости/масштабирования (`K`).
4. `ui/state.py`: `UIState`, пресеты, входные параметры симуляции, хранение текущих значений виджетов.
5. `ui/panels.py`: панели управления (скорость, угол/спин/направление, мяч, поверхность/слои, трение, `Запустить`, `Сравнить`, `Самопроверка`, `Масштаб`).
6. `physics/model.py`: верхнеуровневый `PhysicsModel` (единый интерфейс `reset`, `step`, сбор метрик/выходов).
7. `physics/surface.py`: дискретная поверхность, эквивалентные параметры слоев, расчёт внутренних сил и граничных условий.
8. `physics/contact.py`: вычисление нормальной и касательной сил (Fn/Ft), stick-slip и накопление `slip`-метрик.
9. `physics/ball.py`: динамика мяча (x, y, v, ω), интегратор semi-implicit Euler и ограничение энергии (`clampRebound`-логика).
10. `physics/spikes.py`: упрощенная модель шипов (локальный наклон и влияние на трение/контакт).
11. `physics/post.py`: короткий пост-контактный полёт и критерий завершения отрыва.
12. `render/renderer.py`: `Renderer` (главный `draw`), отрисовка мяча/поверхности/разреза.
13. `render/overlays.py`: оверлеи: векторы `Fn`, `Ft`, `v`, `v_rel`; контактное пятно и карта давления; отрисовка стрелок/меток.
14. `render/graphs.py`: графики по времени (`Fn`, `Ft`, `def`, `slip`, `ω`, `vx`, `vy`), единый формат истории.
15. `validation/checks.py`: `valid`, `checks` и sanity-проверки параметров перед симуляцией.
16. `utils/math.py`: вспомогательные функции (`clamp`, `sign`, геометрия/ограничения).

Ключевое требование к реализации: численное ядро не зависит от `arcade` (рендер/UI только читают состояние и используют готовые метрики/массивы для отрисовки).
Для структур параметров/состояния и типизации планируется использовать стандартные модули `dataclasses` и `typing`.

### 3. Математическая модель (подробно)

#### 3.1. Система координат и геометрия

- 2D-разрез: ось `x` — горизонтально, `y` — вертикально вверх.
- Мяч — окружность радиуса `r`, масса `m`, момент инерции `I = if * m * r^2`.
- Поверхность — дискретная линия по `x` на равномерной сетке узлов: `x_i` от `-hw` до `+hw`, `n` узлов.
- Каждый узел имеет вертикальные и горизонтальные степени свободы: `u_y[i]`, `u_x[i]`, скорости `v_y[i]`, `v_x[i]`.

#### 3.2. Слои поверхности и эквивалентные параметры

- Вводимые слои: толщина `t`, жесткости `k_n`, `k_t`, демпфирование `c_n`, `c_t`, трения `mu_s`, `mu_k`, режим шипов (`none/out/in`), `k_sh`, `h`.
- Плотности материалов берутся из таблицы (например дерево/карбон/губка/топшит/полимер/бетон); для неизвестных материалов используется дефолтная плотность.
- Плотность шипов (`pd`) фиксирована во внутренней модели и не вводится пользователем.
Эквивалентная нормальная/касательная жесткость по слоям (гармоническое сложение):

1. `cN_i = (t_i / 1000) / k_n_i`, `cT_i = (t_i / 1000) / k_t_i`.
2. `sN = Σ cN_i`, `sT = Σ cT_i`.
3. `k_eqN = 1 / sN`, `k_eqT = 1 / sT`.

Взвешивание параметров верхних слоев экспонентой `exp(-i * p)` (слой ближе к поверхности влияет сильнее):

1. `mu_s`, `mu_k`, `c_n`, `k_n`, `k_t`, `c_t` берутся как взвешенные средние.
2. Глобальный множитель трения `frMul` умножает `mu_s`, `mu_k` и затем значения ограничиваются диапазонами.
3. `mu_s` и `mu_k` дополнительно ограничиваются на уровне модели (`mu_k <= mu_s`).

- Масса узла: `m_node = dx * depth * Σ (ρ_i * t_i)`.

#### 3.3. Внутренние силы в поверхности (узлы)

Для каждого узла:

1. Вертикально: базовая пружина/демпфер к основанию `F_y += -k_by * u_y - c_by * v_y`, плюс связь с соседями `k_ly*(u_{i±1}-u_i) + c_ly*(v_{i±1}-v_i)`.
2. Горизонтально: базовая пружина/демпфер к основанию `F_x += -k_bx * u_x - c_bx * v_x`, плюс связь с соседями `k_lx*(u_{i±1}-u_i) + c_lx*(v_{i±1}-v_i)`.
3. Крайние узлы усиливаются (edge stiffening): дополнительные `k` и `c` на первых/последних `2..3` узлах.

Уравнения движения узла:

- `m_node * a_y = F_y`, `m_node * a_x = F_x`.
- Интегрирование: semi-implicit Euler (сначала скорости, затем позиции).
- Клиппинг скоростей и смещений (верхние/нижние границы устойчивости).

#### 3.4. Модель шипов (упрощенная)

- Для верхнего слоя с режимом `out/in` вводится локальный наклон `θ` и скорость наклона `θ_dot`.
- Динамика наклона (без явной массы, условно единичная): `θ¨ = f_th - k_sh * θ - c_sh * θ_dot`, где `f_th = pg * F_t * scale`, а `scale` зависит от направления шипов; `θ` ограничен `[-θ_max, θ_max]`.
- Наклон повышает `mu_s`, `mu_k` и усиливает `F_t` в контакте.
- Параметры `k_sh`, `c_sh` вычисляются из пользовательского `k_sh` верхнего слоя и внутренних коэффициентов модели.

#### 3.5. Контакт: нормальная сила

Для каждого узла в пределах радиуса мяча:

1. Геометрия: `dd = x_ball - x_i`, `y_ball_surface = y_ball - sqrt(r^2 - dd^2)`, `penetration δ = u_y[i] - y_ball_surface`.
2. Если `δ > 0`, контакт активен.
3. Нормальная сила: `F_n = k_c * δ^p + c_c * max(0, v_rel_n)`, где `p = 1.35`, `v_rel_n = -(v_ball_y - v_node_y)`.
4. `F_n` ограничивается сверху защитным капом для устойчивости.
5. Демпфирование по нормали действует только при сближении (через `max(0, v_rel_n)`).

#### 3.6. Контакт: касательная сила и трение

- Относительная скорость: `v_rel_t = (v_ball_x - ω * r) - v_node_x`.
- Накапливается «stick» смещение: `s += v_rel_t * dt`.
- Пробная сила: `F_t_trial = -k_s * s`.
- Критерий: если `|F_t_trial| <= μ_s * F_n`, то «stick» и `F_t = F_t_trial`, иначе «slip»: `F_t = -sign(v_rel_t) * μ_k * F_n`, а `s` пересчитывается из `F_t`.
- Время проскальзывания суммируется для метрик.

#### 3.7. Динамика мяча

- Переменные: `x, y, v_x, v_y, ω`.
- Ускорения: `a_x = F_t / m`, `a_y = F_n / m - g`, `ω_dot = -(F_t * r) / I`.
- Интегратор: semi-implicit Euler.
- Энергия ограничивается сверху: `KE <= KE0 * 0.999`.

#### 3.8. Критерий отрыва

- Контакт считается завершенным, если одновременно выполнены: контакт отсутствует `N` шагов подряд, `y > r + margin`, `v_y` выше порога.

#### 3.9. Пост-контактный полет

- Симуляция короткого полета на `SIM_POST_DUR`: `v_y` уменьшается на `g * dt`, `ω` затухает `ω *= (1 - k_spin * dt)`, положение обновляется по скорости.

#### 3.10. Энергетическая защита (clampRebound)

- Если итоговая кинетическая энергия превышает начальную, скорости масштабируются вниз.
- Если одновременно выросли `|v|` и `|ω|`, применяется приоритетное ограничение одной компоненты, чтобы не было одновременного увеличения.
- Это обеспечивает отсутствие искусственного «разгона» отскока.

#### 3.11. Преподлет и центр удара

Время подлета рассчитывается из масштаба и скорости:

1. `t_pre = min(Tmax, PX / (scale * speed))`.
2. `x0 = -v_x * t_pre`.
3. `y0 = r - v_y * t_pre + 0.5 * g * t_pre^2`.

- Это обеспечивает удар в центре по `x` при визуализации.

#### 3.12. Свободные параметры модели

- Мяч: `r`, `m`, `if`, `k`, `c`.
- Входные: `speed`, `angle`, `spin`, `spinDir`.
- Слои: `t, k_n, c_n, k_t, c_t, mu_s, mu_k, p, k_sh, h`.
- Глобальный множитель трения `frMul`.
- Качество: `normal/high` (влияет на `dt`, `n`).
- Константы устойчивости и масштабирования вынесены в `K`.

#### 3.13. Граничные условия

- Края сетки усилены (`edge stiffening`), что имитирует зафиксированное основание и предотвращает «расползание».
- Вертикальные и касательные смещения узлов ограничены верхними пределами.
- Давление и контактное пятно рассчитываются локально на основе активных узлов.

#### 3.14. Методы для последующих расчетов

Можно расширять:

- Добавить более точную модель демпфирования (частотно-зависимую) для слоев.
- Ввести нелинейность в `k_n`, `k_t` по глубине деформации.
- Расширить расчет до 3D с осевым распределением контакта.
- Ввести явное разделение «преподлета» и «контакта» для чистого `contact_time`.

#### 3.15. Допущения модели

- Контакт рассчитывается в 2D-разрезе, без полноценной 3D-геометрии.
- Внутренняя деформация поверхности моделируется пружинно-демпферной сетью, а не FEM.
- Аэродинамика не моделируется, кроме простого затухания spin.
- Нормальная сила учитывает «Hertz-like» нелинейность, но без точной калибровки по материалу.
- Контактное пятно оценивается по активным узлам, без полноценного решения распределения давления.

### 4. Визуализация (целевое поведение в Arcade)

- Мяч рисуется по физическим `(x, y)` и вращается по фазе `phi += ω dt`.
- На мяче есть метка и стрелка направления spin.
- Слои поверхности рисуются статически, цвета и контуры слоев видимы.
- Динамическая деформация поверхности **не визуализируется** (есть только расчет внутри модели).
- Оверлеи: векторы `Fn`, `Ft`, `v`, `v_rel`; контактное пятно и карта давления; графики по времени `Fn`, `Ft`, `def`, `slip`, `ω`, `vx`, `vy`.
- Ракетка центрируется по X, расположена у нижней части окна с учетом толщины слоев.

### 5. UI и параметры (целевое поведение)

- Панели ввода: столкновение (скорость, угол, spin, направление), мяч (полый/сплошной, радиус, жесткость, демпфирование), поверхность (редактируемый список слоев).
- Панель действий: `Запустить`, `Сравнить`, `Самопроверка`, `Масштаб` (циклически 120% → 100% → 85% → 70% → 55%).
- Слои: добавить слой, удалить слой (кнопка в каждой строке), перемещение вверх/вниз, пресеты `classic`, `inv`, `hard`, подробные заголовки с единицами измерения.
- Управление трением: `mu_s`, `mu_k` в таблице слоев, глобальный множитель трения (числовой ввод + ползунок).
- Анимация: скорость по умолчанию `0.005`, ползунок `0.001..0.100`.
- Клик по области окна: пауза/продолжение.

### 6. Выходные данные

- Числовые метрики: `v_out`, `omega_out`, `angle_out`, `contact_time` (включает подлет), `max_def`, `max_shift`, `slip_share`, `energy_loss`, `J_n`, `J_t`.
- Графики: `Fn`, `Ft`, `def`, `slip`, `ω`, `vx`, `vy`.
- Сравнение до 3 прогонов.

### 7. Критерии приемки (соответствуют текущей реализации)

- Угол входа влияет на траекторию и итоговый угол выхода.
- Вращение визуально видно и соответствует знаку `ω`.
- Ракетка по X центрирована, удар по центру кадра.
- Динамическая деформация поверхности не рисуется.
- Анимация читаема и замедлена (default `0.005`).
- `hard` пресет не даёт отскока назад по X при дефолтных значениях.

### 8. Ограничения и допущения

- Только 2D-разрез; нет полноценной 3D-модели.
- Динамическая деформация рассчитывается, но не визуализируется.
- Контактное время включает фазу подлета (в текущей реализации).
- Допускаются численные калибровки (кап энергии/сил) для стабильности.
