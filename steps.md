# Пошаговый план реализации (Python 3.14 + arcade) по `plan_py.md`

Ниже — план “от нуля до готового приложения”, с промежуточными точками тестирования/документации и возможностью остановиться, чтобы человек мог проверить решения и внести правки до продолжения.

## 0. Предварительные договоренности
1. Код физики держим полностью независимым от `arcade` (рендер/UI читают готовые состояния/массивы/метрики).
2. Структура модулей должна соответствовать смысловым разделам: `physics/`, `render/`, `ui/`, `config/`, `validation/`, `utils/`.
3. Версия Python: `3.14`. `arcade`: последняя версия, установленная в глобальном Python (мы не используем виртуальное окружение).
4. Неизменными по смыслу остаются: физика (контакт, трение, шипы, энергия/отрыв), интерфейсные элементы/поведение и визуальные идеи (вращение, слои в разрезе, оверлеи, графики).

## 7. Физика контакта: нормаль Fn и трение Ft (stick-slip)
1. Создать `physics/contact.py`:
   - вычисление для каждого активного узла:
     - геометрия `dd`, вычисление `y_ball_surface`, penetration `δ = u_y - y_ball_surface`
     - активация контакта только при `δ > 0`
     - нормальная сила `F_n = k_c * δ^p + c_c * max(0, v_rel_n)` + защитный cap
   - вычисление касательной скорости `v_rel_t`
   - накопление stick-смещения `s`
   - критерий stick-slip: `|F_t_trial| <= mu_s * F_n` => stick, иначе slip через `mu_k`
   - суммирование времени проскальзывания для `slip_share`
2. Убедиться, что контакт работает согласованно с массивами поверхности (активные узлы определяются по радиусу мяча).
3. Обязательная защита от численной нестабильности:
   - cap силы по нормали/касательной
   - аккуратный расчет `sqrt(r^2-dd^2)` с защитой от округлений.
4. Документировать:
   - `python_arcade_sim/docs/STEP_07_CONTACT_MODEL.md`.
5. Тесты:
   - несколько сценариев с заведомо “малой” скоростью: контакт кратко появляется и исчезает без `NaN`
   - при `mu_s=mu_k=0` трение не должно давать большой касательной силы.

## 8. Физика мяча и интегратор
1. В `physics/ball.py` реализовать динамику мяча:
   - `a_x = F_t / m`
   - `a_y = F_n / m - g`
   - `omega_dot = -(F_t * r) / I`
2. Интегратор:
   - semi-implicit Euler (сначала скорости, потом позиции)
3. Пост-контактный полёт и критерий отрыва:
   - post на `SIM_POST_DUR`
   - критерий завершения: контакт отсутствует `N` шагов подряд, `y > r + margin`, `v_y` выше порога
4. Энергетическая защита `clampRebound`:
   - если KE растёт сверх начальной — масштабировать скорости вниз
   - если одновременно выросли `|v|` и `|ω|` — приоритетное ограничение одной компоненты
5. Документировать:
   - `python_arcade_sim/docs/STEP_08_BALL_AND_INTEGRATION.md`.
6. Тесты:
   - на простом случае (например угол малый, spin 0) — траектория должна быть физически правдоподобной и не уходить в разлет.
