# Анализ физики контакта мяча с поверхностью

## Проблемы текущей Python реализации

### 1. Неправильная модель контакта
В JS реализации используется **упрощенная модель с одной точкой контакта**:
- `depth` — глубина погружения мяча
- `fn = kN * depth^1.35 + cN * depthRate` — нормальная сила
- `vRel = vt - omega * radius` — относительная скорость проскальзывания
- `stickDisp += vRel * dt` — накопление смещения
- `ft = -kT * stickDisp` (stick) или `ft = -sign(vRel) * muK * fn` (slip)

В Python реализации — **сеть узлов с деформацией**, что избыточно и создаёт проблемы:
- Узлы не успевают деформироваться за малое время контакта
- Силы распределяются неправильно
- Мяч "залипает" или отскакивает непредсказуемо

### 2. Неправильный расчёт трения
В JS:
```javascript
stickDisp += vRel * dt;
const ftTrial = -kT * stickDisp;
const fStaticMax = muS * fn;
if (|ftTrial| <= fStaticMax) {
    ft = ftTrial;  // stick
} else {
    ft = -sign(vRel) * muK * fn;  // slip
    stickDisp = -ft / kT;
}
```

В Python — сложная модель со stick-slip, но работает неправильно.

### 3. Угловое ускорение
В JS:
```javascript
alpha = -(ft * radius) / I;
omega += alpha * dt;
```

В Python то же, но `ft` рассчитывается неправильно.

## Ключевые инсайты из JS

### 1. Модель одного контакта (lumped parameter)
Вместо сети узлов используется **единая точка контакта** с параметрами:
- `depth` — глубина деформации мяча
- `depthRate` — скорость деформации
- `stickDisp` — накопленное касательное смещение

### 2. Эквивалентные параметры поверхности
```javascript
// Гармоническое сложение жёсткостей
let kAcc = 0;
layers.forEach((lay, idx) => {
    const thick = lay.thicknessMm / 1000;
    const weight = 1 / (1 + idx * 0.6);
    const k = (lay.stiffness / thick) * weight * 0.00045;
    kAcc += 1 / k;
});
const kN = 1 / kAcc;  // эквивалентная нормальная жёсткость
const kT = kN * 0.32;  // касательная жёсткость
```

### 3. Простая интеграция
```javascript
// Ускорения
const an = (fn / m) - g;
const at = ft / m;
const alpha = -(ft * radius) / I;

// Интегрирование
vn += an * dt;
vt += at * dt;
omega += alpha * dt;

// Деформация
depthRate += -vn * dt * 900;  // коэффициент связи
depth += depthRate * dt;
depth = max(0, depth);
```

## Предлагаемая новая модель для Python

### 1. Отказ от сети узлов
Использовать **модель одной точки контакта**:
- `overlap` — перекрытие мяча с поверхностью
- `overlap_rate` — скорость перекрытия
- `stick_disp` — касательное смещение

### 2. Правильный расчёт сил
```python
# Нормальная сила (закон Герца с демпфированием)
fn = kc * overlap^1.35 + cc * max(0, overlap_rate)

# Относительная скорость проскальзывания
v_rel = v_t - omega * r

# Касательная сила (stick-slip)
stick_disp += v_rel * dt
ft_trial = -kt * stick_disp
ft_max = mu_s * fn

if abs(ft_trial) <= ft_max:
    ft = ft_trial  # stick
else:
    ft = -sign(v_rel) * mu_k * fn  # slip
    stick_disp = -ft / kt
```

### 3. Правильное интегрирование
```python
# Ускорения
a_n = fn / m - g
a_t = ft / m
alpha = -(ft * r) / I

# Скорости
v_n += a_n * dt
v_t += a_t * dt
omega += alpha * dt

# Деформация (связь с нормальной скоростью)
overlap_rate += -v_n * dt * K_coupling
overlap += overlap_rate * dt
overlap = max(0, overlap)
```

## Тестовые сценарии для проверки

### Сценарий 1: Вертикальное падение с вращением
- Вход: `v_x ≈ 0`, `v_y = -5 м/с`, `omega = ±100 рад/с`
- Ожидание: После отскока `v_x` пропорционально `omega`

### Сценарий 2: Удар под 45° без вращения
- Вход: `angle = 45°`, `omega = 0`
- Ожидание: Угол отскока > 45°, появляется `omega > 0` (по часовой)

### Сценарий 3: Удар под 45° с верхним вращением
- Вход: `angle = 45°`, `omega = +100 рад/с`
- Ожидание: Угол отскока < 45°, `v_x` растёт

### Сценарий 4: Удар под 45° с нижним вращением
- Вход: `angle = 45°`, `omega = -100 рад/с`
- Ожидание: Угол отскока >> 45°, `v_x` падает, `omega` может смениться

## Результаты тестов lumped model (2026-03-25)

### Проблема 1: Знак omega
При ударе под 45° без вращения omega становится положительным (против часовой), а должно быть отрицательным (по часовой).

**Причина:** При контакте мяч движется вправо (v_x > 0), трение направлено влево (ft < 0). Сила влево создаёт момент:
```
alpha = -(ft * r) / I
```
При ft < 0: alpha = -(-|ft| * r) / I > 0 — ускорение ПРОТИВ часовой!

**Но это неправильно!** При ударе мяча о поверхность, трение должно создавать вращение ПО часовой (omega < 0).

**Где ошибка?** В физике: сила трения направлена ПРОТИВ относительной скорости. При v_x > 0, v_rel > 0, ft < 0. Но момент от этой силы:
- Сила приложена к НИЖНЕЙ точке мяча
- Сила направлена ВЛЕВО
- Это создаёт вращение ПРОТИВ часовой!

Стоп... это правильно! При ударе мяча о стол, трение действительно создаёт вращение против часовой стрелки (верхнее вращение).

**Вывод:** Физика правильная, но мои ожидания были неверны! При ударе без вращения мяч получает ВЕРХНЕЕ вращение (omega > 0, против часовой).

### Проблема 2: Угол отскока
При верхнем вращении (omega > 0) угол отскока должен быть < 45°, но получается > 45°.

**Причина:** Верхнее вращение (omega > 0) создаёт v_rel = v_x - omega * r < v_x. Это уменьшает трение, но не меняет знак.

Нужно проверить: при большом omega > 0, v_rel может стать < 0, тогда трение сменит направление!

### Проблема 3: Скорость выхода
Скорость выхода 15 м/с при входе 10 м/с — это нарушение сохранения энергии!

**Причина:** Модель не учитывает потерю энергии правильно. Нужно добавить диссипацию.

## План исправлений

3. **Калибровать параметры:** kc, cc, kt нужно подобрать для реалистичного контакта
