# Физика контакта мяча с поверхностью — исправленная модель

## Проблема оригинальной модели

Оригинальная модель с сеткой узлов имела фундаментальные проблемы:

1. **Сеть узлов не успевала реагировать** на быстрое изменение скоростей мяча при контакте
2. **`active_nodes = 0`** — узлы не детектировались в контакте, даже когда мяч "пробивал" поверхность
3. **Силы трения вычислялись неправильно** — `ft = 0` даже при наличии контакта
4. **`clamp_rebound_priority` обрезал `v_x`** неправильно, используя `ke_initial` из начала симуляции, а не из момента контакта

## Решение: Lumped Parameter Model

Вместо сети узлов используется **единая точка контакта** с параметрами:
- `overlap` — глубина деформации мяча, м
- `overlap_rate` — скорость деформации, м/с
- `stick_disp` — касательное смещение для трения, м (ограничено ±5 мм)

### Физическая модель

**1. Нормальная сила (закон Герца с демпфированием):**
```python
fn = kc * overlap^1.35 + cc * max(0, overlap_rate)
```

**2. Касательная сила (stick-slip трение):**
```python
v_rel = v_x - omega * r  # Относительная скорость проскальзывания
stick_disp += v_rel * dt
stick_disp = clamp(stick_disp, -0.005, 0.005)  # Ограничение 5 мм
ft_trial = -kt * stick_disp
ft_max = mu_s * fn

if abs(ft_trial) <= ft_max:
    ft = ft_trial  # stick
else:
    ft = -sign(v_rel) * mu_k * fn  # slip
    stick_disp = -ft / kt
```

**3. Угловое ускорение (ИСПРАВЛЕНО):**
```python
# ft < 0 (влево) в нижней точке создаёт вращение ПО часовой (alpha < 0)
alpha = (ft * radius) / I  # Знак ПРАВИЛЬНЫЙ (был минус, стал плюс)
```

**4. Энергетическая защита:**
- `ke_initial` вычисляется в **момент начала контакта**, а не в начале симуляции
- `clamp_rebound_priority` вызывается на каждом суб-шаге для ограничения энергии

**5. Суб-степпинг:**
- Каждый шаг контакта разбивается на `CONTACT_SUBSTEPS = 20` подшагов
- Это обеспечивает стабильность интегрирования при больших силах

## Ключевые исправления

1. **Знак момента силы:** `alpha = (ft * r) / I` вместо `alpha = -(ft * r) / I`
   - Сила влево в нижней точке мяча создаёт вращение ПО часовой (omega < 0)

2. **`ke_initial` в момент контакта:** Сохраняется при переходе в CONTACT, а не в reset()

3. **Ограничение `stick_disp`:** ±5 мм предотвращает нереалистично большие силы трения

4. **Позиционная коррекция:** При переходе в контакт мяч устанавливается точно на поверхность

5. **Контакт активен при `v_y < 0`:** Даже если `overlap = 0`, контакт считается активным, пока мяч движется вниз
