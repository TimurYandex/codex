# Ball Impact Simulator (Python + Arcade)

Реалистичный симулятор удара мяча о многослойную поверхность.

## Быстрый старт

```bash
cd python_arcade_sim
python main.py
```

## Управление

### Мышь
- **ЛКМ** по окну: Пауза/продолжение
- **ЛКМ** по кнопкам: Выполнение действий

### Клавиатура
- **Пробел**: Пауза/продолжение
- **R**: Перезапуск
- **G**: Переключить графики
- **O**: Переключить оверлеи
- **1/2/3**: Пресеты поверхности (classic/inv/hard)
- **Escape**: Выход

### Кнопки действий
- **Run/Pause**: Запуск/пауза симуляции
- **Compare**: Режим сравнения (до 3 прогонов)
- **Self Test**: Самопроверка
- **Zoom**: Переключение масштаба (120% → 100% → 85% → 70% → 55%)

## Физическая модель

### Фазы симуляции
1. **PREFLIGHT**: Мяч движется по параболе к поверхности
2. **CONTACT**: Взаимодействие с поверхностью (деформация, трение, шипы)
3. **POST**: Пост-контактный полёт с гравитацией

### Компоненты
- **Мяч**: Твёрдое тело с вращением (5 степеней свободы)
- **Поверхность**: Многослойная структура с эквивалентными параметрами
- **Контакт**: Lumped parameter model с законом Герца и stick-slip трением
- **Шипы**: Наклон θ влияет на трение

### Lumped Parameter Model

Вместо дискретной сети узлов используется единая точка контакта с параметрами:
- `overlap` — глубина деформации мяча
- `overlap_rate` — скорость деформации
- `stick_disp` — касательное смещение (ограничено ±5 мм)

**Преимущества:**
- Стабильная физика при больших скоростях
- Правильное взаимодействие вращения с поступательным движением
- Суб-степпинг (20 подшагов) для точности интегрирования

См. [`docs/physics_fix.md`](docs/physics_fix.md) для подробного описания.

### Метрики
- `v_out`: Итоговая скорость, м/с
- `omega_out`: Итоговое вращение, рад/с
- `angle_out`: Итоговый угол, градусы
- `contact_time`: Время контакта, мс
- `slip_share`: Доля проскальзывания
- `energy_loss`: Потеря энергии, Дж

## Структура проекта

```
python_arcade_sim/
├── app/           # Главное окно
├── config/        # Константы

│   ├── surface.py        # Поверхность
│   └── sim_types.py      # Типы данных
├── render/        # Рендеринг
│   ├── renderer.py # Базовая отрисовка
│   ├── overlays.py # Векторы, контактное пятно
│   └── graphs.py   # Графики
├── ui/            # Интерфейс
│   ├── state.py   # UIState
│   └── panels.py  # Панели управления
├── validation/    # Валидация
│   └── checks.py  # Проверки параметров
```

### Критерии приемки

- ✅ Угол входа влияет на траекторию и угол выхода
- ✅ Вращение визуально видно и соответствует знаку ω
- ✅ Ракетка по X центрирована, удар по центру кадра
- ✅ Анимация читаема и замедлена (default 0.005)

### Тестовые сценарии

| № | Сценарий | Ожидание | Статус |
|---|----------|----------|--------|
| 1 | Вертикальное падение с вращением | angle ≈ 90° | ✅ |
| 2 | Удар под 35° без вращения | angle > 35°, omega < 0 | ✅ |
| 3 | Удар под 35° с верхним вращением | angle < 35° | ✅ |
| 4 | Удар под 35° с нижним вращением | angle >> 35° | ✅ |
